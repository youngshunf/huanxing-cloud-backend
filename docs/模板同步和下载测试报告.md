## 三、修复的问题

### 1. CRUD 方法返回值问题

**问题**: `marketplace_sync_log_dao.create()` 返回 `None`

**修复**: 修改返回类型为 `MarketplaceSyncLog`

```python
# 修复前
async def create(self, db: AsyncSession, obj: CreateMarketplaceSyncLogParam) -> None:
    await self.create_model(db, obj)

# 修复后
async def create(self, db: AsyncSession, obj: CreateMarketplaceSyncLogParam) -> MarketplaceSyncLog:
    return await self.create_model(db, obj)
```

### 2. 字典传递问题

**问题**: 传递 `dict` 给期望 Pydantic 模型的 CRUD 方法

**修复**: 使用 Pydantic 模型包装数据

```python
# 修复前
await marketplace_sync_log_dao.create(db, {
    'sync_type': 'github',
    'status': 'in_progress',
    'started_at': datetime.now()
})

# 修复后
await marketplace_sync_log_dao.create(db, CreateMarketplaceSyncLogParam(
    sync_type='github',
    status='in_progress',
    started_at=datetime.now()
))
```

### 3. 字段名不一致

**问题**: 
- `_parse_template_yaml` 返回 `app_id`，但 schema 期望 `template_id`
- `version_data` 使用 `package_path`，但模型字段是 `package_url`

**修复**: 统一字段名

```python
# 修复前
app_data = {
    'app_id': app_id,
    ...
}

version_data = {
    'package_path': package_info['package_path'],
    ...
}

# 修复后
app_data = {
    'template_id': app_id,
    'template_type': 'agent',
    ...
}

version_data = {
    'package_url': package_info['package_path'],
    ...
}
```

### 4. 缺少 mark_all_not_latest 方法

**问题**: `marketplace_template_version_dao` 缺少 `mark_all_not_latest` 方法

**修复**: 添加方法

```python
async def mark_all_not_latest(self, db: AsyncSession, template_id: str) -> int:
    """将应用的所有版本标记为非最新"""
    return await self.update_model_by_column(db, {'is_latest': False}, template_id=template_id)
```
