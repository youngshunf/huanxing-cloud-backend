# 代码生成使用说明

## 🚀 快速开始

### 一键生成（推荐）

```bash
# 方式一：从 SQL 文件生成（自动建表 + 生成全部代码）
uv run fba codegen generate --sql-file backend/sql/tables/xxx.sql --app myapp --execute

# 方式二：从已有数据库表生成
uv run fba codegen generate --table user,role --app myapp --execute
```

> **`--sql-file` 模式会自动检测表是否存在，不存在则执行 SQL 建表**，无需手动建表。

## 📋 参数说明

| 参数 | 说明 | 示例 |
|------|------|------|
| `--sql-file` | SQL 建表文件路径（与 --table 二选一） | `backend/sql/tables/xxx.sql` |
| `--table` | 数据库中已存在的表名，逗号分隔 | `user,role,menu` |
| `--app` | 应用/模块名称 | `huanxing`, `admin` |
| `--execute` | 自动执行生成的菜单和字典 SQL | 添加此参数即启用 |
| `--schema` | 数据库 schema（默认 `public`） | `public` |

## 📝 SQL 文件编写规范

以 PostgreSQL 为例，字典字段注释需遵循固定格式：

```sql
CREATE TABLE "public"."app_version" (
  "id"          bigserial PRIMARY KEY,
  "status"      varchar(20) NOT NULL DEFAULT 'draft',
  "platform"    varchar(30) NOT NULL,
  "created_time" timestamptz(6) NOT NULL DEFAULT NOW(),
  "updated_time" timestamptz(6)
);

-- 表注释
COMMENT ON TABLE "public"."app_version" IS '应用版本表';

-- 普通字段
COMMENT ON COLUMN "public"."app_version"."id" IS '主键 ID';

-- ✅ 字典字段注释格式: 字段名 (value:label:color/value:label:color)
COMMENT ON COLUMN "public"."app_version"."status" IS '状态 (draft:草稿:blue/published:已发布:green/deprecated:已废弃:orange)';
COMMENT ON COLUMN "public"."app_version"."platform" IS '平台 (darwin:macOS/win32:Windows/linux:Linux)';
```

### 字典注释格式要点

- 使用**英文括号** `()` 或中文括号 `（）` 均可
- 每个选项格式: `value:label` 或 `value:label:color`
- 选项之间用 `/` 分隔
- 括号前的文字作为字典类型名称（如"状态"、"平台"）

> ⚠️ **错误示例**: `'状态：0-待办 1-进行中'`  
> ✅ **正确示例**: `'状态 (0:待办:blue/1:进行中:orange/2:已完成:green)'`

## 📂 生成的文件结构

```
后端代码 (backend/app/<app>/):
├── model/codegen_test_task.py          # SQLAlchemy 模型
├── schema/codegen_test_task.py         # Pydantic Schema
├── crud/crud_codegen_test_task.py      # CRUD 操作
├── service/codegen_test_task_service.py # 业务逻辑
└── api/
    ├── router.py                        # 路由注册（支持多 scope）
    └── v1/
        ├── admin/codegen_test_task.py   # 管理端 API（JWT + RBAC）
        ├── app/codegen_test_task.py     # 用户端 API（JWT，用户数据隔离）
        ├── agent/codegen_test_task.py   # Agent API（API Key + X-User-Id）
        └── open/codegen_test_task.py    # 公开 API（无认证，只读）

前端代码 (clound-frontend/apps/web-antdv-next/src/):
├── views/<app>/<table>/index.vue       # 主页面
├── views/<app>/<table>/data.ts         # 表格和表单配置
└── api/<app>/<table>.ts                # API 接口定义

SQL 文件 (backend/sql/generated/):
├── <table>_menu.sql                     # 菜单和权限 SQL
└── <table>_dict.sql                     # 字典数据 SQL
```

## 🔐 多 Scope API 说明

代码生成器支持 4 种 API scope，每种有不同的认证方式：

| Scope | 路径 | 认证方式 | 特点 |
|-------|------|----------|------|
| **admin** | `/api/v1/<app>/` | JWT + RBAC + Permission | 管理端，完整权限控制 |
| **app** | `/api/v1/<app>/app/` | JWT only | 用户端，`request.user.id` 数据隔离 |
| **agent** | `/api/v1/<app>/agent/` | Agent Key | 通过 `X-User-Id` Header 传入用户 |
| **open** | `/api/v1/<app>/open/` | 无 | 公开只读，仅 GET 接口 |

### 配置默认 scope

在 `config.toml` 中设置：

```toml
[backend]
# 逗号分隔，支持多个 scope 同时生成
default_api_scope = "admin"              # 仅管理端
default_api_scope = "admin,app"          # 管理端 + 用户端
default_api_scope = "admin,app,agent,open" # 全部 scope
```

## ⚙️ 配置文件

路径：`backend/plugin/code_generator/config.toml`

```toml
[paths]
frontend_dir = "../huanxing-cloud-frontend"  # 前端项目目录
menu_sql_dir = "backend/sql/generated"       # SQL 输出目录

[generation]
existing_file_behavior = "skip"       # skip / overwrite
auto_execute_menu_sql = false         # 是否自动执行菜单 SQL
auto_execute_dict_sql = false         # 是否自动执行字典 SQL

[backend]
default_api_scope = "admin"           # 默认 API scope

[dict]
auto_dict_patterns = ["status", "type", "state", "level", "category"]
```

## 💡 完整使用示例

### 示例：从 SQL 文件生成任务管理模块

```bash
# 1. 编写建表 SQL（含字典字段注释）
cat > backend/sql/tables/task.sql << 'EOF'
CREATE TABLE "public"."task" (
  "id"           bigserial PRIMARY KEY,
  "user_id"      bigint NOT NULL,
  "title"        varchar(200) NOT NULL,
  "status"       smallint DEFAULT 0,
  "type"         varchar(32) DEFAULT 'normal',
  "created_time" timestamp DEFAULT now(),
  "updated_time" timestamp
);

COMMENT ON TABLE "public"."task" IS '任务表';
COMMENT ON COLUMN "public"."task"."status" IS '状态 (0:待办:blue/1:进行中:orange/2:已完成:green/3:已取消:red)';
COMMENT ON COLUMN "public"."task"."type" IS '类型 (normal:普通:blue/urgent:紧急:red/scheduled:计划:green)';
EOF

# 2. 一键生成全部代码（自动建表 + 生成代码 + 执行SQL）
uv run fba codegen generate --sql-file backend/sql/tables/task.sql --app myapp --execute
```

### 从已有表生成

```bash
# 批量生成（表已存在于数据库）
uv run fba codegen generate --table user,role,permission --app admin --execute
```

## 📚 其他可用命令

| 命令 | 说明 |
|------|------|
| `fba codegen generate --sql-file ... --app ...` | 从 SQL 文件一键生成（推荐） |
| `fba codegen generate --table ... --app ...` | 从已有表一键生成 |
| `fba codegen frontend` | 仅生成前端代码 |
| `fba codegen menu` | 仅生成菜单 SQL |
| `fba codegen import` + `fba codegen` | 分步交互式生成 |

## ✅ 检查清单

**生成前：**
- [ ] SQL 文件使用正确的字典注释格式 `(value:label:color/...)`
- [ ] 表和字段名使用 `snake_case` 格式
- [ ] 配置文件 `config.toml` 中路径和 scope 设置正确

**生成后：**
- [ ] 前端文件已生成（`views/<app>/<table>/`）
- [ ] 后端代码结构完整（model/schema/crud/service/api）
- [ ] API 按 scope 分目录（admin/app/agent/open）
- [ ] 菜单 SQL 和字典 SQL 已生成到 `sql/generated/`
