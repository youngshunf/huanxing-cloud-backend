## 关键问题与解决方案

### 问题 1: 数据库字段名不一致
**问题**：数据库表使用 `create_time/update_time`，但 SQLAlchemy 的 DateTimeMixin 使用 `created_time/updated_time`

**解决方案**：
- 统一使用 `created_time/updated_time` 作为标准字段名
- 重建数据库表以匹配模型定义
- 创建表重建脚本 `tests/recreate_hasn_task.py` 和 `tests/recreate_hasn_sessions.py`

### 问题 2: hasn_sessions 主键不匹配
**问题**：数据库表使用 `session_id` 作为主键，但模型使用 `id`

**解决方案**：
- 修改 `HasnSessions` 模型，将主键改为 `session_id: Mapped[str]`
- 更新所有引用该字段的代码
- 修复 CRUD 层排序字段

### 问题 3: API 路由配置错误
**问题**：路由前缀配置导致 404 错误

**解决方案**：
- 移除子路由的前缀，依赖父路由的 `/api/v1/hasn/app` 前缀
- 确保路由注册顺序正确
