# S3 存储配置指南

## 问题说明

头像上传功能需要配置 S3 存储才能正常工作。如果未配置 S3 存储，上传头像时会返回错误：

```
S3 存储配置不存在。请先在管理后台配置 S3 存储（系统管理 -> S3存储管理），或使用兼容 S3 的本地存储服务（如 MinIO）。
```

## 解决方案

### 方案一：使用 MinIO（推荐用于开发环境）

MinIO 是一个兼容 S3 API 的开源对象存储服务，适合本地开发和测试。

#### 1. 安装并启动 MinIO

**使用 Docker（推荐）：**

```bash
docker run -d \
  --name minio \
  -p 9000:9000 \
  -p 9001:9001 \
  -e MINIO_ROOT_USER=minioadmin \
  -e MINIO_ROOT_PASSWORD=minioadmin \
  -v ~/minio/data:/data \
  quay.io/minio/minio server /data --console-address ":9001"
```

**或使用 Homebrew（macOS）：**

```bash
brew install minio/stable/minio
minio server ~/minio/data --console-address ":9001"
```

#### 2. 访问 MinIO 控制台

打开浏览器访问：http://localhost:9001

- 用户名：`minioadmin`
- 密码：`minioadmin`

#### 3. 创建存储桶

在 MinIO 控制台中：
1. 点击左侧菜单 "Buckets"
2. 点击 "Create Bucket"
3. 输入桶名称：`huanxing`
4. 点击 "Create Bucket"

#### 4. 在唤星管理后台配置 S3 存储

登录唤星管理后台，进入 **系统管理 -> S3存储管理**，添加以下配置：

- **存储名称**：本地 MinIO
- **终端节点**：`http://localhost:9000`
- **访问密钥**：`minioadmin`
- **密钥**：`minioadmin`
- **存储桶**：`huanxing`
- **前缀**：`avatars`（可选）
- **区域**：`us-east-1`（MinIO 默认）
- **CDN域名**：留空（本地开发不需要）

### 方案二：使用云服务商的 S3 存储

#### 阿里云 OSS

1. 登录阿里云控制台
2. 开通对象存储 OSS 服务
3. 创建 Bucket
4. 获取 AccessKey 和 SecretKey
5. 在唤星管理后台配置：
   - **终端节点**：`https://oss-cn-hangzhou.aliyuncs.com`（根据实际区域调整）
   - **访问密钥**：你的 AccessKeyId
   - **密钥**：你的 AccessKeySecret
   - **存储桶**：你的 Bucket 名称
   - **区域**：`oss-cn-hangzhou`（根据实际区域调整）
   - **CDN域名**：如果配置了 CDN，填写 CDN 域名

#### 腾讯云 COS

1. 登录腾讯云控制台
2. 开通对象存储 COS 服务
3. 创建存储桶
4. 获取 SecretId 和 SecretKey
5. 在唤星管理后台配置：
   - **终端节点**：`https://cos.ap-guangzhou.myqcloud.com`（根据实际区域调整）
   - **访问密钥**：你的 SecretId
   - **密钥**：你的 SecretKey
   - **存储桶**：你的存储桶名称
   - **区域**：`ap-guangzhou`（根据实际区域调整）

#### 七牛云 Kodo

1. 登录七牛云控制台
2. 开通对象存储服务
3. 创建空间
4. 获取 AccessKey 和 SecretKey
5. 在唤星管理后台配置：
   - **终端节点**：`https://s3-cn-east-1.qiniucs.com`（根据实际区域调整）
   - **访问密钥**：你的 AccessKey
   - **密钥**：你的 SecretKey
   - **存储桶**：你的空间名称
   - **CDN域名**：你的 CDN 加速域名

## 验证配置

配置完成后，尝试上传头像：

1. 登录唤星 WebUI
2. 点击右上角用户头像
3. 点击"新建分身"
4. 点击头像区域上传图片
5. 如果配置正确，应该能成功上传并显示头像

## 常见问题

### 1. 上传时返回 405 错误

**原因**：S3 endpoint 配置不正确，或者使用了不支持的 HTTP 方法。

**解决方法**：
- 检查 endpoint 是否正确（MinIO 默认是 `http://localhost:9000`）
- 确保 MinIO 服务正在运行
- 检查防火墙是否阻止了连接

### 2. 上传时返回 403 错误

**原因**：AccessKey 或 SecretKey 不正确，或者没有写入权限。

**解决方法**：
- 检查 AccessKey 和 SecretKey 是否正确
- 确保存储桶的访问策略允许写入
- 在 MinIO 控制台中，检查用户权限

### 3. 上传成功但无法访问图片

**原因**：存储桶的访问策略不允许公开读取。

**解决方法**：
- 在 MinIO 控制台中，进入 Buckets -> huanxing -> Access Policy
- 设置为 "Public" 或添加自定义策略允许公开读取

### 4. 生产环境建议

- 使用云服务商的 S3 存储（阿里云 OSS、腾讯云 COS 等）
- 配置 CDN 加速域名以提高访问速度
- 定期备份重要数据
- 设置合理的访问策略和权限控制
- 启用 HTTPS 加密传输

## 技术细节

### OpenDAL

唤星使用 [OpenDAL](https://github.com/apache/opendal) 库来统一访问各种存储服务。OpenDAL 支持：

- AWS S3
- 阿里云 OSS
- 腾讯云 COS
- 七牛云 Kodo
- MinIO
- 以及其他兼容 S3 API 的存储服务

### 代码位置

- 头像上传 API：`backend/app/admin/api/v1/sys/user.py` 的 `upload_user_avatar` 函数
- S3 存储模型：`backend/plugin/s3/model/storage.py`
- S3 存储 CRUD：`backend/plugin/s3/crud/storage.py`
