## API 端点

### Session API

**Base URL**: `/api/v1/hasn/app`

- `POST /sessions/upsert` - 创建或更新 Session
- `POST /sessions/{session_id}/summary` - 更新 Session 摘要
- `POST /sessions/{session_id}/close` - 关闭 Session
- `GET /sessions` - 查询 Session 列表
- `GET /sessions/{session_id}/messages` - 查询 Session 消息
- `POST /sessions/{session_id}/messages` - 发送消息

### Task API

**Base URL**: `/api/v1/hasn/app`

- `GET /tasks` - 查询任务列表
- `POST /tasks` - 创建任务
- `GET /tasks/{task_id}` - 查询任务详情
- `PUT /tasks/{task_id}` - 更新任务
- `DELETE /tasks/{task_id}` - 删除任务
- `POST /tasks/{task_id}/enable` - 启用任务
- `POST /tasks/{task_id}/disable` - 禁用任务

## 故障排查

### API 返回 404
- 检查路由注册：确认 router.py 中正确注册了路由
- 检查路由前缀：app router 的 prefix 是 `/api/v1/hasn/app`
- 重启后端服务

### WebSocket 测试失败
- 检查 hasn-node 编译：`cargo check --workspace`
- 检查测试文件：`apps/daemon/tests/ws_session_events.rs`
- 运行单个测试：`cargo test test_push_session_message_received`

### 数据库连接失败
- 检查 PostgreSQL 服务：`pg_isready -h localhost -p 15432`
- 检查 .env 配置：`DATABASE_PORT=15432`
- 检查数据库名称：`DATABASE_SCHEMA=huanxing`
