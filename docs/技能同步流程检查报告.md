## 一、技能来源和同步方式

### 1. ClawHub 同步
- **来源**: https://clawhub.ai
- **目标目录**: `huanxing-hub/clawhub/{owner}/{slug}/`
- **同步方式**:
  - API 获取技能元数据
  - 下载 zip 文件并解压到本地
  - 翻译 name 和 description
  - 保存到数据库
- **触发方式**:
  - 手动触发: `POST /api/v1/marketplace/admin/sync/clawhub`
  - 定时任务（可选）
- **当前状态**: ✅ 已同步 2 个技能

### 2. GitHub 仓库同步 (skills/)
- **来源**: https://github.com/youngshunf/huanxing-hub.git
- **目标目录**: `huanxing-hub/skills/{category}/{slug}/`
- **同步方式**:
  - Git pull 拉取最新代码
  - 扫描 skills/ 目录下的所有技能
  - 解析 manifest.yaml 文件
  - 翻译 name 和 description
  - 保存到数据库
- **触发方式**:
  - 手动触发: `POST /api/v1/marketplace/admin/sync/github`
  - Webhook: `POST /api/v1/marketplace/webhook/github/skills`
  - 当 huanxing-hub 仓库 push 时自动触发
- **当前状态**: ✅ 已同步 143 个技能

### 3. 自己整理的技能 (huanxing/community)
- **来源**: 手动添加或从其他来源整理
- **目标目录**: 与 GitHub 仓库同步相同
- **同步方式**: 与 GitHub 仓库同步相同
- **当前状态**: ✅ 已同步 3 个技能

## 二、自动同步流程

### GitHub Webhook 自动同步

```
GitHub 仓库 push
    ↓
GitHub Webhook 触发
    ↓
POST /api/v1/marketplace/webhook/github/skills
    ↓
验证签名（可选）
    ↓
检查是否有 skills/ 目录变更
    ↓
触发 github_sync_service.sync_from_github()
    ↓
1. Git pull 拉取最新代码
2. 扫描 skills/ 目录
3. 解析 manifest.yaml
4. 调用翻译服务
5. 保存到数据库
6. 更新同步日志
    ↓
返回同步结果
```

### 服务器部署配置

**生产环境需要配置**:

1. **环境变量** (`.env`):
   ```bash
   # huanxing-hub 本地路径
   HUANXING_HUB_LOCAL_PATH='/data/huanxing-hub'
   
   # GitHub Webhook Secret（生产环境必须配置）
   GITHUB_WEBHOOK_SECRET='your-webhook-secret'
   
   # LLM 翻译服务
   LLM_API_BASE_URL='https://fuyou.cool'
   LLM_API_KEY='sk-your-api-key'
   TRANSLATION_MODEL='gpt-5.4-mini'
   
   # ClawHub API
   CLAWHUB_API_URL='https://clawhub.ai/api/v1'
   ```

2. **GitHub Webhook 配置**:
   - URL: `https://your-domain.com/api/v1/marketplace/webhook/github/skills`
   - Content type: `application/json`
   - Secret: 与 `GITHUB_WEBHOOK_SECRET` 一致
   - Events: `push`

3. **服务器 Git 配置**:
   ```bash
   # 克隆 huanxing-hub 仓库
   cd /data
   git clone https://github.com/youngshunf/huanxing-hub.git
   
   # 配置 Git（避免 pull 时需要输入凭据）
   cd huanxing-hub
   git config credential.helper store
   ```
