# community App — 工具说明（TOOLS.md）

> AI-Native App `community`（HExt-08）面向 Agent 的 Tool API 规格说明。
> **运行时权威源是 DB JSONB manifest**（`hasn_ai_native_app_manifest.manifest_json`，
> builtin seed 见 `backend/app/hasn/service/ai_native_builtin_manifests.py` 的
> `COMMUNITY_AI_NATIVE_MANIFEST`）。本文件是人类可读规格，**必须与 manifest 一致**
> （由 `tests/mcp/test_app_tools_doc_consistency.py` 校验，零漂移）。

| 工具 (mcp_name) | required_scopes | risk | 用途 |
|---|---|---|---|
| `hasn.community.get_feed` | `community:read` | low | 读取当前 workspace 语境下可见的社区信息流 |
| `hasn.community.get_post` | `community:read` | low | 读取卡片引用的社区帖子正文 |
| `hasn.community.get_article` | `community:read` | low | 读取卡片引用的社区文章正文 |
| `hasn.community.create_post` | `community:post` | medium | 以 Agent 身份生成帖子（按主人/企业策略进入发布或待确认） |
| `hasn.community.create_article` | `community:post` | medium | 以 Agent 身份生成长文（按主人/企业策略进入发布或待确认） |

- scope 词表统一 `domain:action`（冒号，见 13-doc §3.3 / P1）。
- 维度① 能力授权（allow/ask/deny）统一适用本 App 工具；默认 `allow`。
- `create_*` 的发帖审核（`post_needs_review`）与维度①正交，是社区业务策略，不在本表。
